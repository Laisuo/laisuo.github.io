注意 在网上给的教程中，bucket: , access_key: ,secret_key: ,后面都有{}。 我按照提示输入了个人信息后，在运行hexo qiniu sync之后一直给我提示：
FATAL Something’s wrong. Maybe you can find the solution here: http://hexo.io/docs/troubleshooting.html
TypeError: Key must be a buffer
折腾了半天后，找到node/modules/hexo-qiniu-sync目录下的README.md文档，发现不需要{}。删除{}之后，同步正常。

plugins: qiniu: offline: false sync: true bucket: {your qiniu bucket name} access_key: {your AK} secret_key: {your SK} dirPrefix: urlPrefix: {your qiniu domain,for example:http://xxxxxx.xxx.clouddn.com} local_dir: static update_exist: true image: folder: images extend: js: folder: js css: folder: css

网站制作过程中遇到的问题

图片插入

问题描述：采用七牛云做为图床之后，之前使用的本地图片引用方式全部失效，导致一开始写的博文中，图片无法正常加载！
* 在根目录下的_config.yml中，将post_asset_folder: false修改为post_asset_folder: true。运行hexo g之后，在public对应的文件夹中会将需要的图片复制过去（如果该命令是false，图片不会被复制）。但是hexo d之后，图片仍然无法正常显示。
* 在安装不同插件过程中发现一直出现同样的WARN。比如运行npm install hexo-asset-image –save，出现npm WARN SKIPPING OPTIONAL DEPENDENCY: fsevents@1.1.3 (node_modules):
npm WARN SKIPPING OPTIONAL DEPENDENCY: Unsupported platform for fsevents@1.1.3: wanted {“os”:“darwin”,“arch”:“any”} (current: {“os”:“win32”,“arch”:“x64”}). After seaching, it seems this is not the reason tha cause my current problem.
* 打开任意public中生成的网页，查看网页源代码。找到图片对应的html语句

，其中的链接无法定位到正确的图片，即http://yoursite.com/2018/01/11/Material-Characterization-Technique-XPS/Material-Characterization-Technique-XPS/XPS.png出了问题。
* 打开Github，进入对应的博客位置，打开index.html，发现链接与上面一样。 * 猜测是http://yoursite.com无法直接定位到正确的图片导致。
* 最终解决办法：根据以上探索，发现将图片备份到七牛云是更好的做法。（1）如果将图片在本地保持，在hexo g的过程中，在public文件夹中会复制相同的图片，同时在github上也会复制一份图片，相当于同样的图片会在三个不同的位置进行备份，极大的占据了不必要的空间。（2）当图片数量较大是，以后的博客上传可能会出现问题，速度问题和github免费容量问题。（3）通过阅读一些博客，发现正确的图片引用方式是采用标签插件，而不是markdown语句。

公式

问题描述：使用Mathjax工具来插入公式，在本地预览正常(F6)，但当布局到网站上时无法正常显示！
* 阅读了Dreamland 的博客发现在站点配置文件上出了问题。需要布置math:等语句 * 进一步阅读DanielLaah 的博客发现在博客书写上也出现了问题。尽管在MarkdownPad2编辑器中，添加了设置 (click Tools > Options > Advanced > HTML Head Editor）,但是当将md文件布置成html文件时，上述的设置并没有起作用（也不知道为什么）。所以在md文档的开头添加一段命令即可。 * 下次撰写md文档，凡是需要调用mathjax，都需要再该文档的开头添加一段命令，这样在布置html文件时才可以避免出现公式无法正常显示。 * 最后的矛盾：md文件本地预览和在线预览，总有一个无法正常。因为在md语音中_有斜体的意义，而在mathjax语音中_表示小标。为了解决这个矛盾，在md文本中，可以采用_来表示斜体，但是这个规定似乎并没有被mathjax接受。因此，当采用_表示斜体时，md文件本地预览(F6)正常；当采用_表示斜体时，md文件在线预览正常。
